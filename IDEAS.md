### 1.3. Изображения и даташиты – автоматизация поиска

Сейчас фото и даташиты приходится искать вручную. Хочется автоматизировать, но с умом.

#### 🔍 Гибридный поиск с ручным подтверждением

1. Пользователь в диалоге редактирования вводит название детали (или MPN).
2. Нажимает кнопку **«Найти в интернете»**.
3. Программа параллельно опрашивает несколько источников и показывает окно с результатами в виде карточек:
   - миниатюра фото (если есть);
   - источник (`datasheet.su`, `chipdip.ru` …);
   - тип (изображение / PDF).
4. Пользователь кликает на подходящий вариант – программа скачивает файл и автоматически заполняет поле `image_path` или `datasheet_path` в форме.
5. Если ничего не найдено – кнопка **«Добавить вручную»** (стандартный выбор файла).

#### 💾 Кеширование и запоминание

- Если для детали `NE555` пользователь выбрал фото с `chipdip.ru`, программа запоминает этот выбор. При следующем добавлении такой же детали (или похожей) может предложить тот же источник автоматически.

#### 🧩 Система поисковых плагинов (пользовательские источники)

Чтобы не зависеть от изменений сайтов и дать пользователю свободу:

- В меню `Сервис → Управление источниками` открывается окно со списком активных поисковиков.
- Можно добавлять свои источники через простую форму:
  - **Название**: `Поиск на ЧипДип`
  - **URL-шаблон**: `https://www.chipdip.ru/search?searchtext={query}`
  - **Тип**: `открыть в браузере` (пока так; в будущем – `скачать PDF`).
- Программа подставляет вместо `{query}` название детали и открывает сайт.
- Пользователи могут делиться готовыми настройками (импорт/экспорт JSON).

## 🌐 2. Веб-интерфейс (SPA)

### 2.1. Текущий эксперимент (`spa-experiment`)

Уже реализовано в ветке `spa-experiment`:
- Дерево категорий загружается динамически.
- Клик по категории фильтрует таблицу без перезагрузки.
- Кнопки «Все / В наличии / Мало / Нет» работают через AJAX.
- Поиск по таблице в реальном времени.
- Обновление статистики автоматически.

### 2.2. Полноценное SPA с маршрутизацией

**План на будущее:**  
Создать отдельное приложение на базе **Vue 3** (или React) с использованием:
- **Vue Router** – для страниц (главная, о программе, настройки, статистика).
- **Pinia** (или Redux) – для хранения состояния деталей, категорий, фильтров.
- **Vite** – для быстрой сборки.

**Структура:**
- `frontend-spa/` – отдельная папка в проекте.
- Компоненты: `PartTable`, `CategoryTree`, `FilterBar`, `StatsPanel`.
- Production-сборка – статика, которую можно отдавать через FastAPI (будет единый порт).

**Зачем:**  
Веб-интерфейс станет ещё более отзывчивым и удобным, появится возможность легко добавлять новые страницы (например, графики, настройки профиля).

## 🔍 3. Источники данных (с учётом России)

| Источник | Тип данных | Способ доступа | Примечание |
|----------|------------|----------------|-------------|
| `datasheet.su` | даташиты (PDF) | поиск по названию + прямая ссылка | Огромная база (~48 млн), можно использовать как основной источник. |
| `chipdip.ru` | фото, цена, наличие | парсинг или прямой поиск | Самый популярный российский магазин. |
| `platan.ru` | фото, цена, наличие | официальный API | Удобно, не нужно парсить. |
| `radel.ru` (Радиодетали РФ) | старые/советские детали | ручной поиск или парсинг | Хорош для ретро-компонентов. |
| `yandex.ru/images` | фото | открытие браузера по шаблону | Резервный вариант, если на профильных сайтах нет картинки. |

**Приоритет:** сначала встроить простые открыватели браузера (через систему плагинов), затем добавить прямое скачивание для `datasheet.su`, потом – для магазинов.

## 🤖 4. Искусственный интеллект (футуристично, но интересно)

Все идеи ниже – **для дальнейших экспериментов**, не для ближайших версий.

| Возможность | Описание | Реализация (российские аналоги) |
|-------------|----------|--------------------------------|
| **Распознавание детали по фото** | Сфотографировал микросхему – программа определяет её модель. | Yandex Vision (обученная модель) или локальная YOLO. |
| **OCR с фото** | Сфотографировал маркировку – текст в поле «Наименование». | Yandex Vision, Tesseract, EasyOCR. |
| **Голосовой ввод** | Нажал на микрофон, сказал: «Конденсатор 100 мкФ 25В, 10 штук» – форма заполнилась. | Yandex SpeechKit, Vosk (локальный). |
| **Автозаполнение параметров из даташита** | Программа скачала PDF, ИИ прочитал его и вытащил корпус, напряжение, ток. | Yandex GPT (или ChatGPT API, если доступен). |
| **Рекомендация аналогов** | «BC547 нет, но есть 2N3904 – можете использовать». | Требует базы знаний или анализа характеристик через LLM. |

**Ограничения:** API западных сервисов могут быть недоступны, но Яндекс предоставляет хорошие альтернативы. Многие модели можно запускать локально (без интернета).

## 📝 6. Примеры сценариев использования (как это будет работать)

### Сценарий А: Добавление новой детали с поиском фото

1. Пользователь открывает диалог «Добавить деталь», вводит название `NE555`.
2. Нажимает кнопку **«Найти в интернете»**.
3. Появляется окно с результатами: слева – фото с `chipdip.ru`, справа – даташит с `datasheet.su`.
4. Пользователь кликает по фото – оно скачивается и путь вставляется в поле `image_path`.
5. Кликает по даташиту – PDF скачивается, путь вставляется в поле `datasheet_path`.
6. Остальные поля заполняет вручную (или позже дообучим ИИ). Нажимает «Сохранить».

### Сценарий Б: Массовое перемещение деталей

1. В таблице пользователь выделяет 10 резисторов (удерживая `Ctrl`).
2. Нажимает правой кнопкой → **«Изменить выбранные»** (или кнопку на панели).
3. В диалоге меняет поле `Место` с `Ящик 1` на `Ящик 3`.
4. Нажимает `ОК` – все 10 деталей обновлены за секунду.

### Сценарий В: Добавление нового источника поиска (например, AliExpress)

1. Пользователь заходит в меню `Сервис → Управление источниками`.
2. Нажимает **«Добавить»**.
3. Вводит:
   - Название: `AliExpress`
   - URL: `https://www.aliexpress.com/wholesale?SearchText={query}`
   - Тип: `открыть в браузере`.
4. Сохраняет.
5. Теперь в контекстном меню детали появился пункт «Искать на AliExpress».
