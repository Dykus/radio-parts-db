# Журнал изменений (Changelog)

Все значимые изменения в проекте `RadioPartsDB` будут документироваться в этом файле.
Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
версионирование — на [Semantic Versioning](https://semver.org/lang/ru/).

## [0.20.0-beta] — 2026-05-26

### 🚀 Добавлено (SPA-эксперимент)
- **Одностраничное приложение (SPA)**: Веб-интерфейс полностью переписан на чистом JavaScript без перезагрузок страницы.
- **Динамическое дерево категорий**: Категории загружаются с бэкенда, при клике таблица мгновенно фильтруется по выбранной категории.
- **Фильтры без перезагрузки**: Кнопки «Все», «В наличии», «Мало», «Нет» работают через AJAX, таблица обновляется на лету.
- **Живой поиск**: Фильтрация таблицы по всем полям (наименование, тип, корпус, место, статус) по мере ввода текста.
- **Визуальное выделение активной категории**: Выбранная категория подсвечивается синим фоном.
- **Обновление статистики**: Статистика (количество деталей, общая стоимость) пересчитывается автоматически при изменении данных.

### 🐛 Исправлено (Web Client)
- Устранены проблемы с загрузкой категорий и деталей из-за неполных данных в API (добавлена обработка отсутствующих полей).

### 🔧 Изменено
- Веб-клиент (`frontend/index.html`) теперь полностью самодостаточен и не требует дополнительных сборщиков.

---

## [0.19.1] — 2026-05-26

### 🐛 Исправлено
- **Валидация Pydantic V2**: Устранена ошибка `ResponseValidationError` при отдаче данных из API. Реализован патч словарей БД для гарантированного наличия всех опциональных полей.
- **Синтаксис моделей**: Переход на `ConfigDict` вместо устаревшего `class Config`.
- **Запуск проекта**: `run.bat` теперь автоматически устанавливает зависимости и запускает все три компонента (Backend, Desktop, Web) без ручного вмешательства.

### ✨ Добавлено
- Полный файл `requirements.txt` с зависимостями для Backend (FastAPI, Uvicorn, Pydantic).
- Автоматическая проверка и создание виртуального окружения при первом запуске.

### 🔧 Изменено
- Версия обновлена до `0.19.1`.

## [0.19.0] — 2026-05-24

### ✨ Добавлено
- **Backend API**: Сервер на FastAPI с полным CRUD, Swagger UI (`/docs`) и CORS.
- **Web Client**: Простой HTML/JS интерфейс, загружающий данные из API.
- **Smart Launcher**: `run.bat` теперь запускает Backend, Desktop и Web одновременно.
- **Безопасный SQLite**: Включён режим `WAL` и `check_same_thread=False` для корректной работы с несколькими подключениями.

### 🐛 Исправлено
- **Pydantic V2 Fix**: Патч словарей БД перед сериализацией для предотвращения `ResponseValidationError`.
- Ошибка `PydanticDeprecatedSince20` (заменено `class Config` на `ConfigDict`).
- Мгновенное закрытие `run.bat` из-за проблем с `activate.bat` и кодировкой.
- Отсутствие полей `category_id`, `image_path` в ответах API вызывало краш валидации.

### 🔧 Изменено
- Ядро `core/database.py` полностью изолировано и переиспользуется в Desktop и API.
- Архитектура переведена в режим `Backend + Frontend` с сохранением совместимости.
- Версия обновлена до `0.19.0` в `config.py`.

### 🎯 Цель
- Подготовка к созданию веб-интерфейса (браузер) и мобильного приложения (Android).
- Единая точка доступа к данным: изменения в Desktop сразу видны в API и наоборот.

## [0.18.0] — 2026-05-24

### Исправлено
- **Логика раскрытия деревьев:** Полностью переписан механизм управления глубиной категорий и навигатора. Заменён ненадёжный метод Qt `expandToDepth()` на рекурсивную функцию `_apply_depth_recursive()` для точного контроля уровней (0, 1, 2, 3, -1).
- **Конфликт интерфейсов:** Устранено дублирование элементов управления в правой панели. Убран выпадающий список глубины, добавлена компактная кнопка `🔄 Глубина` рядом с заголовком навигатора.
- **Исчезновение данных:** Исправлена ошибка, из-за которой центральная таблица могла очищаться при изменении настроек глубины.

### Добавлено
- **Жирный шрифт для категорий:** Категории, содержащие детали (количество > 0), теперь автоматически отображаются жирным шрифтом для быстрой визуальной ориентации.
- **Перетаскивание колонок:** Добавлена возможность менять порядок столбцов таблицы мышкой (Drag-and-Drop заголовков).
- **Сохранение порядка колонок:** Текущая конфигурация столбцов автоматически сохраняется в `window_settings.json` и восстанавливается при следующем запуске.

### 🔧 Изменено и улучшено
- **`.gitignore`:** Обновлён список игнорируемых файлов. Исключены база данных, логи, бэкапы и локальные настройки окна (`window_settings.json`).
- **Документация:** Добавлен файл `LICENSE` (MIT License, автор Dykus). Обновлён `README.md` с разделом о лицензии.
- **Версия:** Обновлена до `0.18.0` в `config.py` и заголовке окна.
