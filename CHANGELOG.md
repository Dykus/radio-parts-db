## [0.21.1] — 2026-05-28

### ✨ Добавлено (Desktop)

- **Экспорт в CSV и Excel** (меню «Файл» → «Экспорт»)
  - Экспортируется текущая отфильтрованная таблица (категория, статус наличия, место хранения, текстовый поиск).

- **Реорганизация верхнего меню**:

- Панель инструментов теперь содержит только основные действия: Добавить, Редактировать, Удалить, Пакетное редактирование, фильтры и поиск.
- Кнопка «📁 Файл» на панели инструментов удалена.

### 📚 Документация

- Обновлён `CHANGELOG.md`, `IDEAS.md`.
- Создан `utils/exporter.py`.

## [0.21.0] — 2026-05-28

### ✨ Добавлено (Desktop)

#### Окно просмотра детали
- **Двойной клик** по строке таблицы открывает немодальное окно «Просмотр детали» (только чтение).
- Отображаются все поля детали, включая размеры конденсаторов, номинал с единицей измерения.
- Изображение (локальное или из интернета) автоматически вписывается в левую область (максимум 400×400) с сохранением пропорций.
- Кнопка **«Увеличить»** открывает отдельное окно с масштабированием **колёсиком мыши** и перетаскиванием.
- Кнопки: **«Редактировать»** (открывает редактор, после сохранения данные обновляются и в окне просмотра, и в главной таблице), **«Печать / PDF»** (печать карточки детали), **«Сохранить фото»**.

#### Пакетное редактирование
- Выделение нескольких деталей (`Ctrl+клик`, `Shift+клик`) → кнопка **«Пакетное редактирование»**.
- Массовое изменение: категория, место хранения, статус, тип детали, производитель, цена (установить, увеличить, уменьшить), количество (установить, добавить, отнять).
- Новые значения типов и производителей автоматически добавляются в словари.

#### Горячие клавиши
| Комбинация | Действие |
|------------|----------|
| `Ctrl+N`   | Новая деталь |
| `Ctrl+E`   | Редактировать выбранную деталь |
| `Del`      | Удалить выбранную деталь (с подтверждением) |
| `Ctrl+F`   | Переместить фокус в поле поиска |
| `F5`       | Обновить все данные |

#### Умная сборка названия
- В диалоге редактирования введите **число** и выберите **единицу измерения** (Ом, кОм, МОм, мкФ, нФ, пФ и т.д.).
- Нажмите **«Собрать название»** – программа сформирует имя в формате: `[номинал][единица] [напряжение/мощность] [корпус]` (например, `10мкФ 25В 10x17`).
- Напряжение (`25V` → `25В`) или мощность (`0.5W` → `0.5 Вт`) извлекаются из последнего сегмента категории.

#### Размеры конденсаторов
- При выборе категории, содержащей **«Конденсатор»**, появляются поля: диаметр (мм), высота (мм), шаг выводов (мм), толщина выводов (мм).
- Размеры отображаются в таблице в колонке «Корпус» в формате `⌀10×17мм`, `шаг 5мм`, `вывод 0.5мм`.

#### Выпадающие списки для типа детали и производителя
- Поля **«Тип детали»** и **«Производитель»** – выпадающие списки с возможностью ввода новых значений.
- Новые значения сохраняются в базу данных и появляются в списке при следующем вводе.

#### Индикатор фото в таблице
- В колонке **«Корпус»** перед текстом отображается 📷, если у детали есть изображение.

#### Кнопка «Сегодня» для даты ревизии
- В диалоге редактирования рядом с полем даты появляется кнопка **«📅 Сегодня»**, устанавливающая текущую дату.

#### Улучшенное сохранение состояния окна
- Программа запоминает не только размер и позицию, но и **развёрнутое состояние** (максимизировано / обычное).

#### Яркое выделение строк в таблице
- Выделенные строки подсвечиваются контрастным синим фоном с белым текстом, видимым на любом цветном фоне строк.

#### Полная русификация контекстного меню
- Во всех текстовых полях, полях ввода комбобоксов, заметках и числовых полях контекстное меню переведено на русский язык.

### 🐛 Исправлено
- Убрано `.0` у целых чисел в нормализованном номинале (было `10.0 Ом`, стало `10 Ом`).
- При выборе детали без фото правая панель очищается (не остаётся предыдущего изображения).
- Исправлена ошибка `TypeError` при работе с `None` в полях размеров.
- Исправлено отсутствие сигнала `double_clicked` в `PartsTableWidget` (двойной клик теперь открывает просмотр).
- Окно программы корректно запоминает развёрнутое состояние, размер и позицию.
- **Сохранение единицы измерения** – теперь при редактировании детали единица (value_unit) корректно сохраняется и восстанавливается.
- **Обновление главного окна после редактирования из окна просмотра** – изменения применяются и таблица обновляется.
- **Масштабирование изображения в окне просмотра** – изображение теперь всегда вписывается в отведённую область (как в правой панели главного окна).

### 🔧 Изменено
- База данных обновлена до схемы **версии 4** (добавлены поля диаметра, высоты, шага выводов, толщины выводов).
- Метод `get_all_parts_filtered` теперь возвращает `image_path` и поля размеров.
- Словари `part_type` и `manufacturer` заполняются начальными значениями при первом запуске.

## [0.20.0-beta] — 2026-05-26

### 🚀 Добавлено (SPA-эксперимент)
- **Одностраничное приложение (SPA)**: Веб-интерфейс полностью переписан на чистом JavaScript без перезагрузок страницы.
- **Динамическое дерево категорий**: Категории загружаются с бэкенда, при клике таблица мгновенно фильтруется по выбранной категории.
- **Фильтры без перезагрузки**: Кнопки «Все», «В наличии», «Мало», «Нет» работают через AJAX, таблица обновляется на лету.
- **Живой поиск**: Фильтрация таблицы по всем полям (наименование, тип, корпус, место, статус) по мере ввода текста.
- **Визуальное выделение активной категории**: Выбранная категория подсвечивается синим фоном.
- **Обновление статистики**: Статистика (количество деталей, общая стоимость) пересчитывается автоматически при изменении данных.

### 🐛 Исправлено (Web Client)
- Устранены проблемы с загрузкой категорий и деталей из-за неполных данных в API (добавлена обработка отсутствующих полей).

### 🔧 Изменено
- Веб-клиент (`frontend/index.html`) теперь полностью самодостаточен и не требует дополнительных сборщиков.

## [0.19.0] — 2026-05-24

### ✨ Добавлено
- **Backend API**: Сервер на FastAPI с полным CRUD, Swagger UI (`/docs`) и CORS.
- **Web Client**: Простой HTML/JS интерфейс, загружающий данные из API.
- **Smart Launcher**: `run.bat` теперь запускает Backend, Desktop и Web одновременно.
- **Безопасный SQLite**: Включён режим `WAL` и `check_same_thread=False` для корректной работы с несколькими подключениями.

### 🐛 Исправлено
- **Pydantic V2 Fix**: Патч словарей БД перед сериализацией для предотвращения `ResponseValidationError`.
- Ошибка `PydanticDeprecatedSince20` (заменено `class Config` на `ConfigDict`).
- Мгновенное закрытие `run.bat` из-за проблем с `activate.bat` и кодировкой.
- Отсутствие полей `category_id`, `image_path` в ответах API вызывало краш валидации.

### 🔧 Изменено
- Ядро `core/database.py` полностью изолировано и переиспользуется в Desktop и API.
- Архитектура переведена в режим `Backend + Frontend` с сохранением совместимости.
- Версия обновлена до `0.19.0` в `config.py`.

## [0.18.0] — 2026-05-24

### Исправлено
- **Логика раскрытия деревьев:** Полностью переписан механизм управления глубиной категорий и навигатора. Заменён ненадёжный метод Qt `expandToDepth()` на рекурсивную функцию `_apply_depth_recursive()`.
- **Конфликт интерфейсов:** Устранено дублирование элементов управления в правой панели.
- **Исчезновение данных:** Исправлена ошибка, из-за которой центральная таблица могла очищаться при изменении настроек глубины.

### Добавлено
- **Жирный шрифт для категорий:** Категории, содержащие детали (>0), отображаются жирным шрифтом.
- **Перетаскивание колонок:** Возможность менять порядок столбцов таблицы мышкой.
- **Сохранение порядка колонок:** Конфигурация столбцов сохраняется и восстанавливается.
