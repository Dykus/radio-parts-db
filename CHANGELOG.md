# Журнал изменений (Changelog)

Все изменения в проекте `radio-parts-db` будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
версионирование — на [Semantic Versioning](https://semver.org/lang/ru/).

## [0.3.0] — 2026-05-20

### 🚀 Добавлено
- **Дерево категорий слева**
  - Иерархическая навигация по категориям компонентов
  - Фильтрация таблицы при клике на папку
  - Автоматическое обновление при добавлении новых категорий
  - Поддержка вложенных категорий (родительские и дочерние)
  
- **Улучшенная работа с категориями**
  - Авто-создание категорий при импорте
  - Привязка компонентов к категориям при добавлении
  - Отображение структуры "Радиоэлемент → Тип → Значение"

### ✨ Изменено
- Обновлена версия приложения до v0.3.0
- Улучшена структура базы данных для поддержки иерархии категорий
- Оптимизирована загрузка данных при фильтрации

### 🐛 Исправлено
- Ошибка импорта типа `QDate` (перенесён из QtGui в QtCore)
- Исправлено заполнение даты ревизии в форме редактирования

---

## [0.2.0] — 2026-05-20

### 🚀 Добавлено
- **Импорт данных из CSV файлов**
  - Автоопределение кодировки (UTF-8, UTF-8-SIG, CP1251, Latin1)
  - Автоопределение разделителя (точка с запятой, запятая, табуляция)
  - Умный маппинг колонок (поддержка русских и английских названий)
  - Поддержка импорта из Memento Database, Excel, Radio Electron DB
  - Очистка HTML-тегов из описаний
  - Парсинг дат в формате DD.MM.YYYY
  - Автоматическое создание категорий при импорте
  - Сопоставление русских статусов (Новое/Б/У/Плохое) с базой данных
  - Объединение нескольких колонок "Значение" и "Ед.изм" в полное наименование
  - Подробное логирование процесса импорта
  - Отчёт о результатах (добавлено/ошибок/пропущено)

### 🔧 Техническое
- Добавлен модуль `utils/importer.py` с функциями импорта
- Реализована функция `guess_encoding()` для определения кодировки
- Добавлена функция `clean_html()` для очистки текста от HTML-тегов
- Добавлена функция `parse_date()` для конвертации дат
- Добавлена функция `map_status()` для сопоставления статусов

## [0.1.1] — 2026-05-20

### 🐛 Исправлено
- **Контрастность текста в таблице**
  - Добавлен чёрный цвет текста на цветной фон
  - Устранена проблема с невидимым текстом на светлых фонах
  
- **Совместимость с PySide6**
  - Заменён устаревший метод `toPyDate()` на `toString("yyyy-MM-dd")`
  - Убрано предупреждение `DeprecationWarning` для `AA_EnableHighDpiScaling`
  
- **Исправлена ошибка импорта**
  - Добавлена отсутствующая функция `guess_encoding()`

### ✨ Изменено
- Улучшена диагностика импорта с подробным выводом в консоль
- Добавлена обработка пропущенных строк с пустыми названиями

## [0.1.0] — 2026-05-20

### 🚀 Добавлено
- **Базовая структура проекта**
  - Модульная архитектура: `core/`, `ui/`, `utils/`, `data/`
  - SQLite база данных с поддержкой таблиц:
    - `parts` (компоненты)
    - `categories` (категории)
    - `dictionaries` (справочники)
    - `projects` (проекты)
    - `project_parts` (связь проектов и компонентов)
    - `parts_archive` (архив удалённых компонентов)
  - Поддержка кроссплатформенности (Windows, Linux, macOS)
  - Конфигурация через переменные окружения
  - Система логирования

- **Пользовательский интерфейс на PySide6**
  - Главное окно с таблицей компонентов
  - Форма добавления/редактирования компонентов
  - Диалог выбора файлов для изображений и даташитов
  - Поле поиска с фильтрацией в реальном времени
  - Статус-бар со сводкой по складу

- **CRUD операции**
  - Добавление новых компонентов
  - Редактирование существующих записей
  - Удаление с автоматической архивацией
  - Получение статистики (всего позиций, стоимость, нет в наличии)

- **Цветовая индикация**
  - 🟢 Зелёный: в наличии (кол-во >= 10)
  - 🟡 Жёлтый: мало (кол-во < 10)
  - 🔴 Красный: нет в наличии или сломано

- **Справочники и категории**
  - Автоматическое создание категорий при импорте
  - Предопределённые списки корпусов (0402, 0603, 0805, и т.д.)
  - Предопределённые места хранения
  - Статусы компонентов (new, used, suspect, broken)

- **Работа с данными**
  - Автобэкап базы данных (настройка)
  - Поддержка относительных путей к файлам
  - Экспорт статистики

### 🔧 Техническое
- Использована БД SQLite (один файл, без зависимостей)
- ORM-подход с использованием словарей
- Контекстные менеджеры для безопасной работы с БД
- Индексы для ускорения поиска

### 📄 Документация
- Добавлен README.md с инструкциями
- Создан шаблон CHANGELOG.md
- Добавлен .gitignore для Python-проектов
- Инструкция по синхронизации через облачные диски
