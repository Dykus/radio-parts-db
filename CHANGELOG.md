# Журнал изменений (Changelog)

Все изменения в проекте `radio-parts-db` будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
версионирование — на [Semantic Versioning](https://semver.org/lang/ru/).

---

## [0.14.0] — 2026-05-22

### 🐛 Исправлено
- **Рекурсивная фильтрация категорий**
  - При выборе родительской категории теперь показываются детали из всех подкатегорий
  - Добавлен метод `_get_all_descendant_ids()` для сбора всех дочерних ID
  - Обновлён `get_all_parts_filtered()` для использования рекурсивной фильтрации
  
- **Сохранение и отображение категорий**
  - Исправлена ошибка, из-за которой поле категории сбрасывалось при повторном открытии диалога
  - Диалог редактирования теперь отображает полный путь категории (например, "Новая / Ещё")
  - Упрощена логика: диалог сам вычисляет и возвращает `category_id`
  - Исправлена ошибка `AttributeError` в `_get_category_path()` (добавлена проверка parent на None)

### 🔧 Улучшено
- **Метод `get_category_part_count_recursive()`** — теперь использует рекурсивный подсчёт для точного отображения счётчиков в дереве категорий

## [0.13.0] — 2026-05-22

### 💾 Добавлено
- **Сохранение параметров интерфейса**

  - Автоматическое сохранение и восстановление ширины всех колонок таблицы
  - Настройки хранятся в `window_settings.json` и применяются сразу при запуске программы

## [0.12.0] — 2026-05-21

### 🎨 Улучшено
- **Новая цветовая схема таблицы**
  - Фон строки зависит от количества: 🟢 >10 шт, 🟡 1-9 шт, 🔴 0 шт
  - Ячейка "Состояние" имеет собственный цвет (Новое=зелёный, Б/У=голубой, Неисправно=красный)
  - Улучшена визуальная навигация: сразу видно наличие и состояние детали

### 🐛 Исправлено
- **Фильтры "В наличии/Мало/Нет"**
  - Исправлена ошибка, из-за которой фильтры не применялись без клика по левой панели
  - Теперь таблица обновляется мгновенно при нажатии на кнопку фильтра

### 🔧 Добавлено
- **Управление таблицей**
  - Добавлена сортировка по клику на заголовки колонок
  - Добавлено ручное изменение ширины колонок мышью (Interactive mode)
  - Скрыта колонка ID из визуального отображения (данные остаются в модели)
- **Контекстное меню навигатора**
  - Добавлено контекстное меню для навигатора по местам (Развернуть/Свернуть всё)
  - Убраны дублирующие кнопки с панели категорий для экономии места

### 🎯 Улучшен интерфейс
- Исправлен порядок элементов в левой панели (кнопки "Все категории" и поиск теперь сверху)
- Исправлено дублирование счётчика при переименовании категорий (использование regex)

## [0.11.0] — 2026-05-21

### 🚀 Добавлено
- **Расширенная навигация по категориям**
  - Добавлена кнопка "📂 Все категории" для быстрого сброса фильтра
  - Реализован рекурсивный подсчёт деталей в категориях (с учётом подкатегорий)
  - Добавлен поиск по категориям с сохранением иерархии (показывает родителей найденных элементов)
  - Реализован Drag-and-Drop для перемещения категорий между родителями с автоматическим обновлением БД
  - Добавлены счётчики количества деталей в скобках рядом с названием категории
  - Добавлено контекстное меню по правому клику: переименовать, добавить подкатегорию, удалить, развернуть/свернуть

### 🔧 Улучшено
- **Синхронизация фильтров**
  - Кнопка "📋 Все" в верхней панели теперь сбрасывает и категорию, и фильтр наличия
  - Исправлено поведение при переключении между фильтрами
- **Отображение правой панели**
  - Исправлено исчезновение навигатора по местам (добавлен вызов `load_tree()` в `_refresh_all()`)
  - Улучшена адаптация под тёмную тему Windows (использование `palette()` вместо жёстких цветов)
  - Ограничена максимальная высота области предпросмотра фото

### 🐛 Исправлено
- Ошибка, из-за которой навигатор по местам оставался пустым после запуска программы
- Проблема с вытеснением дерева мест за пределы окна при большом фото

## [0.10.0] — 2026-05-21

### 🛡 Безопасность и Надежность
- **Система версионирования базы данных**
  - Внедрена таблица `schema_version` для отслеживания состояния схемы
  - Реализован автоматический механизм миграций (обновлений) структуры БД
  - Перед каждым изменением схемы создается автоматический бэкап в папку `data/`
  - Добавлена защита от потери данных при ошибках обновления (транзакции, откат)
  - Программа теперь сама обновляет старые базы данных (перевод статусов на русский) без необходимости их удаления

### 🔧 Улучшено
- **Архитектура кода**
  - Логика создания таблиц перенесена в функции миграций
  - Убраны жесткие проверки (`CHECK constraint`), мешающие развитию схемы
  - Добавлены подробные инструкции в код (`core/database.py`) о том, как безопасно менять структуру базы данных в будущем

---

## [0.9.0] — 2026-05-21

### 🏗 Изменено (Refactoring)
- **Модульная архитектура интерфейса**
  - `main_window.py` разделен на логические компоненты: `dialogs` (окна редактирования) и `widgets` (части интерфейса)
  - Созданы файлы `ui/widgets/parts_table.py`, `ui/widgets/info_panel.py`, `ui/dialogs/part_dialog.py`
  - Улучшена читаемость и поддерживаемость кода

### 🐛 Исправлено
- **Миграция базы данных (Критическое)**
  - Реализован автоматический перевод старых английских статусов (`new`, `broken`) на русские (`Новое`, `Неисправно`) при запуске
  - Убрано жесткое ограничение (`CHECK constraint`), вызывавшее ошибку `IntegrityError` при сохранении новых состояний
  - Добавлен безопасный механизм пересоздания таблицы `parts` с сохранением всех пользовательских данных
- **Фильтрация по месту**
  - Исправлена ошибка, из-за которой клик в навигаторе по местам не фильтровал центральную таблицу
- **Темная тема Windows**
  - Правая панель (фото и навигатор) теперь использует системные цвета (`palette(base)`), вместо жесткого белого фона

### 🚀 Добавлено
- **Локализованные статусы с иконками**
  - Поле "Состояние" теперь содержит русский текст и эмодзи (🛒 Новое, ℹ️ Б/У и т.д.)
  - Выпадающий список сделан нередактируемым для предотвращения опечаток
- **Сохранение позиции окна**
  - Программа запоминает координаты и размер окна при закрытии в файле `window_settings.json`
